### 改了什么行为（视觉层面）

**管理后台** — 对齐 `admin.html`：
- 深色侧栏 + 渐变品牌 Logo「灵」
- 导航按「数据层 / 智能配置 / 质量保障 / 系统」分组，带图标与左侧激活条
- 顶栏面包屑、底部用户区
- 首页改为卡片网格入口

**监控看板** — 对齐 `monitor.html`：
- 深色主题（`#0F172A` 背景）
- 顶部告警横幅、品牌 Header（实时绿点 + 刷新）
- 四列 KPI 卡片（顶部彩色边线）
- 折线图面板 + 满意度详情区

**用户端** — 对齐 `user.html`：
- 顶部导航（品牌 + 对话/模板/帮助 + 用户头像）
- 暖橙配色（`#FFF7ED` 背景）
- 圆角历史侧栏、「+ 新对话」主按钮
- 用户/助手分栏气泡、底部输入区
- Ant Design 主色设为 `#F97316`

### 为什么这样改

将 mockup 中的设计 token 抽到 `ui-shared/theme.css`，三端通过 `AppShell variant` 复用，避免各应用重复写 inline style，也符合架构里「参考 mockup 视觉」的约定。

### 做了哪些验证

- `pnpm --filter @hermes/ui-shared lint` — 通过
- 三端 `tsc --noEmit` — 通过

### 假设

- 本地 `make dev` 或 `pnpm dev` 可正常加载 `@hermes/ui-shared/theme.css`
- Google Fonts 在开发环境可访问（Inter / Merriweather / JetBrains Mono）

### 还剩哪些风险或未验证项

- **Admin 子页面**（数据源、模板等）仍主要用 Ant Design 默认表格/表单，未逐页套用 mockup 的 panel/table 样式
- **用户端**「模板中心」「帮助」导航为占位，尚未接路由
- **监控看板**图表为简化折线，未实现 mockup 中的面积渐变与柱状图细节
- 未在浏览器中做端到端视觉验收，建议本地打开三端对比 mockup 确认

本地预览可运行 `make dev`，分别访问管理后台、监控、用户端路由查看效果。如需继续把 Admin 各子页面（表格、统计卡片等）也细化到 mockup 级别，可以指定优先页面。
