日志已确认：`/` 返回 404。正在添加与 `web-admin` 相同的根路径重定向。


## 问题分析

`GET http://localhost:3003/` 返回 404 是**预期行为**，不是服务没启动。

`web-monitor` 在 `next.config.mjs` 里配置了 `basePath: '/monitor'`，页面实际挂载在 `/monitor` 下，而不是根路径 `/`。Nginx 也是按 `/monitor` 代理的。

### 假设验证（运行时证据）

| 假设 | 结论 | 证据 |
|------|------|------|
| **A** `basePath` 导致 `/` 无路由 | **已确认** | `curl /` → 404；`curl /monitor` → 200；日志 `pathname: "/"` |
| **B** 3003 端口不是 web-monitor | **排除** | 响应头 `X-Powered-By: Next.js` |
| **C** `page.tsx` 缺失或损坏 | **排除** | `/monitor` 正常返回 200 |
| **D** 代理/Nginx 路径被截断 | **排除** | 直连 localhost 同样 404 |
| **D** dev 服务未启动 | **排除** | 端口有响应 |

对比 `web-admin`（`basePath: '/admin'`），它已配置了根路径重定向，而 `web-monitor` 之前缺少同样配置。

## 已做修复

在 `apps/web-monitor/next.config.mjs` 中增加了与 `web-admin` 一致的重定向：`/` → `/monitor`。

访问方式：
- **修复前**：需手动打开 `http://localhost:3003/monitor`
- **修复后**：`http://localhost:3003/` 会自动跳转到 `/monitor`

**注意**：修改 `next.config.mjs` 后需要**重启** web-monitor dev 服务才能生效。

<reproduction_steps>

2. 重新启动：`cd apps/web-monitor && pnpm dev`，或在项目根目录运行 `pnpm dev`
3. 浏览器访问 `http://localhost:3003/`，应自动跳转到 `/monitor` 并显示监控看板
4. 也可直接访问 `http://localhost:3003/monitor` 作为备用入口
5. Press Proceed/Mark as fixed when done.
