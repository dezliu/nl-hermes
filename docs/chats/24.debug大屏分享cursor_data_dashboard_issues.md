已确认：report-service 重启后内存丢失导致 preview 404；从 orchestrator 重新 render 可恢复。正在实现 Gateway 预览回退逻辑。


## 根因（已用运行时证据确认）

| 假设 | 结论 | 证据 |
|------|------|------|
| **F** report-service 内存未命中导致 preview 404 | **已确认** | `preview-mem: 404`，Gateway 首次请求也失败 |
| 回退：从 orchestrator 拉 spec 并 re-render | **已确认** | 日志 `renderOk: true`，Gateway 返回 HTML |

调试日志摘录：

```json
{"message":"preview miss, trying re-render","data":{"initialStatus":404}}
{"message":"re-render fallback","data":{"renderStatus":200,"renderOk":true}}
```

**原因**：`/api/reports/:id/preview` 只查 report-service 内存；服务重启或跨会话后 spec 仅在 orchestrator DB，内存为空 → 404 → `preview_failed`。

## 修复

在 `gateway-api` 增加回退逻辑：

1. 先请求 report-service preview  
2. 若 404，从 orchestrator 读取 spec  
3. 调用 `/v1/reports/render` 重新渲染  
4. 再次请求 preview  

download 端点使用相同回退。

- `apps/gateway-api/src/index.ts` — 新增 `fetchReportPreview` / `ensureReportRendered`
- `docs/progress-board.md`
