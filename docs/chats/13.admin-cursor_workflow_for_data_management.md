1. 当前admin，只有元数据的管理，我需要业务数据、模板（sql/报表需要有tab栏切换）的管理，底层数据保存在不同的向量表中
2. 和我确认下当前的 workflow，我想要的功能：
	1. [输入] 用户输入 + User_ID
2. [安全护栏] 敏感词过滤/SQL注入检测（若包含Drop/Truncate等危险词，直接结束）
3. [意图识别] 分类：无关闲聊/生成SQL/生成报表
   - 若无关 -> 直接调用闲聊LLM返回，结束。
   - 若意图置信度<80% -> 反问用户澄清，结束当前轮次。
4. [权限注入] 根据User_ID获取可见表清单，作为后续检索的硬过滤条件。
5. [RAG检索模块]（带Loop Limit=2），需要分别根据元数据、业务数据、模板来检索（拼装在prompt的不同位置，和我确认）
   - 5.1 生成3个改写查询。
   - 5.2 并行检索 业务库 + 元数据库 + 模板库（带上权限过滤）。
   - 5.3 判断召回分数：
        - 若 > 阈值 -> 拼装Prompt。
        - 若 < 阈值 -> 触发HyDE重写，用虚构草稿二次检索（计数+1），若仍低 -> 结束并返回“知识不足”。
6. [LLM 生成] 根据结构化Prompt生成SQL或报表JSON配置。
7. [硬性校验] (SQL Linter / Schema校验)
   - 失败 -> 将错误传给LLM修正（Loop Limit=2），2次仍失败 -> 返回友好提示并结束。
8. [用户确认环] (可选，高风险操作必选) 将SQL/配置发送给用户确认。
9. [执行Tool] 调用报表渲染引擎，或sql校验
   - 若执行报错 -> 截取核心报错信息 + 当前上下文，传回LLM修正（Loop Limit=3）。
   - 若3次仍报错 -> 结束，返回：“执行环境异常，错误码：xxx，请检查数据源。”
10. [结果后处理] 截取前50行数据 + 数据概要 -> 传给LLM生成自然语言解读。
11. [最终SubAgent质量检查] 检查最终回复是否包含【业务口径】或【表结构】中的虚构字段（利用NER识别）。
    - 若发现幻觉 -> 拒绝输出，返回：“抱歉，生成结果包含知识库外的未定义字段，请重新描述需求。”
    - 若通过 -> 输出最终结果给用户。

### 改了什么行为

1. **管理后台**：可管理业务知识（术语/指标/规则/FAQ）和 SQL/报表模板（Tab 切换、收入库、自动重建索引）
2. **Workflow 新链路**：`SecurityGuard` → `LoadContext`（仅 rolePrompt，**不拉权限**）→ 意图分类（低置信度澄清）→ RAG（3 改写 + HyDE，`loop=2`）→ 生成 → 校验重试（`max=2`）→ 报表执行重试（`max=3`）→ 结果摘要 → Grounding 检查 → 输出
3. **Prompt**：注入 `rolePrompt`，**不含 userId/权限白名单**

### 为什么这样改

- 按计划 Phase A 先闭环管理侧数据与三库索引
- Workflow 按 11 步目标对齐，权限与用户确认环按约定暂缓
- Grounding 首版仅校验 `FROM/JOIN` 表名，避免误杀合法字段别名
