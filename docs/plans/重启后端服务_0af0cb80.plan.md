# 重启 orchestrator 与 metadata-service

项目里这两个服务对应：

| 服务 | 端口 | Docker 服务名 |
|------|------|---------------|
| orchestrator | 4010 | `orchestrator` |
| metadata-service | 4050 | `metadata-service` |

先确认你用的是哪种模式（见 [README.md](README.md)）：

- **Docker 全栈**：`make up` 启动，容器内跑编译后的 `node dist/index.js`
- **本地开发**：`make dev` 启动，Turbo 并行跑 `tsx watch`（改源码会自动重载）

---

## 方式 A：Docker 全栈（`make up`）

### 仅重启（不改代码 / 镜像已是最新）

在项目根目录执行：

```bash
docker compose restart orchestrator metadata-service
```

### 改了代码后重启（需重新构建镜像）

```bash
docker compose up -d --build orchestrator metadata-service
```

### 查看是否启动成功

```bash
curl -s http://localhost:4050/health | jq .
curl -s http://localhost:4010/health | jq .
```

或：

```bash
docker compose logs -f orchestrator metadata-service
```

---

## 方式 B：本地开发（`make dev` / `pnpm dev`）

`make dev` 会通过 Turbo 同时启动所有 app 的 `dev` 脚本（[apps/orchestrator/package.json](apps/orchestrator/package.json) 和 metadata-service 均为 `tsx watch`）。

### 改源码后

- **一般不用手动重启**：保存文件后 `tsx watch` 会自动重启对应进程
- 若改了 **workspace 依赖包**（如 `@hermes/workflow`），需先构建依赖再重启：

```bash
pnpm --filter @hermes/workflow build
# 然后在跑 make dev 的终端 Ctrl+C 后重新：
make dev
```

### 只重启单个服务（不重启全部）

新开终端，在项目根目录：

```bash
# metadata-service
pnpm --filter @hermes/metadata-service dev

# orchestrator（另开终端）
pnpm --filter @hermes/orchestrator dev
```

注意：若 `make dev` 已在跑，同端口会冲突，需先在 Turbo 终端停掉对应进程，或只保留单服务 dev。

### 验证

```bash
make health
```

或单独检查：

```bash
curl -s http://localhost:4050/health
curl -s http://localhost:4010/health
```

## 本次候选入池改动需要注意

你刚改的入池逻辑涉及 **orchestrator** 和 **metadata-service** 两个服务：

```mermaid
flowchart LR
  orchestrator -->|"POST /internal/template-candidates"| metadata_service
  metadata_service --> template_candidates[(template_candidates)]
```

- **Docker 模式**：必须 `--build` 重建两个镜像后再 up，否则仍是旧逻辑
- **本地 dev 模式**：若已改 `@hermes/workflow`，先 `pnpm --filter @hermes/workflow build`，再重启 orchestrator

## 快速对照

| 场景 | 命令 |
|------|------|
| Docker，只重启 | `docker compose restart orchestrator metadata-service` |
| Docker，含代码变更 | `docker compose up -d --build orchestrator metadata-service` |
| 本地 dev，全量重启 | 终端 `Ctrl+C` → `make dev` |
| 本地 dev，单服务 | `pnpm --filter @hermes/metadata-service dev` / `pnpm --filter @hermes/orchestrator dev` |
| 验证 | `make health` 或 curl `:4050/health`、`:4010/health` |
